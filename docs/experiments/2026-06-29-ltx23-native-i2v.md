# LTX 2.3 native I2V smoke test — 2026-06-29

## Goal

Validate that the checked-in ComfyUI I2V graph accepts a clean reference image, preserves its composition, and produces a native audio/video MP4 before spending inference time on a Reddit candidate.

## Configuration

- Workflow: `workflows/04-ltx23-native-i2v-audio-api.json`
- Reference: synthetic 384×224 image with no text, logos, UI, signs or packaging
- Frames: 9
- FPS: 24
- Steps: 4
- I2V strength: 0.92
- Reference compression: 18
- Output codec: H.264 video and AAC audio

## Automated result

- ComfyUI accepted the API graph.
- All required node classes and configured model files were available.
- Output duration: 0.375 seconds.
- Output resolution: 384×224.
- Video codec: H.264.
- Audio codec: AAC.
- Audio stream present: yes.

## Visual result

The first and last frames preserve the reference subject, colors, camera framing, background and blank wall object. No pseudo-subtitles, logos, interface graphics or generated writing were visible in the inspected frames.

## Conclusion

The I2V graph passes the technical and composition-control smoke gate. This supports using I2V as the default LTX 2.3 path instead of unconstrained T2V.

This smoke does not prove meme quality, useful motion, intelligible PT-BR narration or shareability because nine frames are too short for those judgments.
